DiagramEventHandler This class defines operations to handle the following incoming events from the client. In each case the event is a sequence of strings and integers. Each event has a fixed arity. Raising an event in the mapping causes the appropriate diagram element (usually the containing element) to be sent a message with the supplied arguments + the mapping. The diagram element is then responsible for updating the diagram model and using the commands listed above to update the client. Users of the mapping should always raise events and not perform the commands directly. New types of diagram element can handle the events in different ways. New types of diagram mapping can exten-d the interface of events that can be raised.

Edge Straightens the edge by comparing a given waypoint to the next waypoint from the list waypoints, which is supposed to hold the remaining waypoints of this edge. The next waypoint is then moved on the x- or y-axis to the point where it is on the same x- or y-axis as the waypoint before, where the position on the other axis is kept. It is moved on that axis which requires less movement. After that the procedure is repeated on the remaining waypoints. If the list of remaining waypoints is empty the nex cycle will stop the recursion. This operation should not be accessed externally as it is the iterating part od a recursion. It should only be used by the counterpart operation which will do the initialisation of the recursion.
