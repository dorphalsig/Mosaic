Meta Modelling

Applied Meta-Modeling

XModeler is based on a self-describing meta-model called XCore. Everything in XModeler is an instance of something from XCore (even XCore itself). The XModeler is based on a language-based approach to meta-modelling. Click on the button below to see the an XModeler-based book describing a language-based approach to meta-modelling.

Meta Tooling

XCore

The XModeler is a collection of tools based on a meta-model called XCore. You can extend and change the XModeler tool-set and define your own tools. The documentation for the whole of XModeler including the source code is provided as a collection of web-pages accessible from the button below.

Meta Programming

XOCL

XModeler is written in a language called XOCL. The language has been designed to make writing XModeler applications straightforward and is supported by XModeler tools (that are also written in XOCL). You can easily extend XOCL to support your own programming abstractions and even define complete domain-specific languages. Click on the button below to find out more about XOCL.

Meta Levels

XOCL

Meta-data is a key feature of modern system design. The XModeler allows data, types and meta-type information to be mixed within a single application. This means that models can be executable, and applications can reflect on their own definitions via access to type-level information. XModeler is the basis of a project that integrates meta-modelling with executable-modelling. Click on the button below for more information.
